# 基于差分演化的混合无线传感器网络覆盖控制优化（DE_ALGO）

## 1. 这个项目实现了什么

这是一个用于**混合无线传感器网络（Hybrid WSN）覆盖优化**的 Python 项目，核心目标是：

- 在给定监测区域内，固定一部分静态节点；
- 允许一部分移动节点调整位置；
- 通过差分演化算法（DE / JDE）搜索移动节点的新坐标；
- 尽可能提高区域覆盖率，同时控制移动代价（移动距离）。

项目提供了两种使用方式：

- 图形界面方式（Tkinter）：可视化参数设置、运行优化、展示前后布局；
- 脚本方式：批量跑一组参数并输出指标，便于实验复现。

## 2. 项目结构与各文件作用

### 2.1 根目录核心文件

- `main.py`：程序入口，启动 GUI 主窗口。
- `ui_main.py`：主界面（菜单、参数设置、算子设置、开始优化、查看布局与结果文件）。
- `ui_params.py`：参数设置弹窗（网络来源、移动比例、DE 参数、输入文件选择）。
- `ui_operator.py`：变异算子选择弹窗（DE/Best/1、DE/Rand/1 等）。
- `ui_result.py`：结果展示窗口；调用算法并绘制"初始/优化后"对比图与性能指标。
- `ui_coords.py`：单独查看初始或优化后节点坐标布局。

### 2.2 算法与数据核心模块（`core/`）

- `core/params.py`：
  - 定义全局常量（监测区域、网格大小、感知半径等）；
  - 定义数据结构（`Sensor`、`GlobalState`）；
  - 负责网络构造（从文件读取或随机生成）；
  - 负责覆盖率计算与网格覆盖判定。
- `core/de_algo.py`：
  - 差分演化主流程（初始化、变异/交叉、选择、迭代）；
  - 支持基本 DE 和 JDE 两种运行入口；
  - 支持多种 DE 变异策略（共 6 种，每种均有基本 DE 版本和 JDE 自适应版本）；
  - 包含后处理：减少不必要移动、交换匹配降低移动距离；
  - 输出结果到 `result/output.txt` 与 `result/coordinates.txt`。
- `core/data_utils.py`：`data/` 与 `result/` 目录管理、输入文件复制等工具函数。
- `core/__init__.py`：包初始化文件（当前为空）。

### 2.3 其他目录

- `data/`：输入网络文件与样例（如 `grid60.txt`、`grid100.txt`、随机数据文件）。
- `result/`：算法运行结果输出（覆盖率、收敛代数、坐标等）。
- `script/validate.py`：命令行验证脚本，固定参数分别跑 DE 与 JDE 并打印指标字典。

## 3. 算法核心实现细节（基于当前代码）

### 3.1 问题建模

- 监测区域在 `core/params.py` 中固定为矩形范围；
- 将区域离散为规则网格（`GRID=28`，总网格数 `GRID_NUM`）；
- 覆盖率定义为：`被覆盖网格数 / 总网格数`；
- 染色体编码长度为 `2 * mobile_num`，按 `[x1, y1, x2, y2, ...]` 表示所有移动节点目标位置。

### 3.2 初始化阶段

`DEAlgorithm.initialize()` 中主要做了：

- 根据移动比例计算移动节点数与静态节点数；
- 读取网络文件或随机生成网络；
- 为种群随机生成初始染色体（每个移动节点随机坐标）；
- 预计算静态节点覆盖集合，后续评估时复用；
- 计算初始覆盖率 `initialcoverage`。

### 3.3 适应度函数

代码中存在多个评估函数：

- `evaluate()`：纯覆盖率目标（JDE 使用）；
- `evaluate_coverage_move_distance()`：覆盖率 + 移动距离折衷（`run_simple_de()` 默认使用）；
- `evaluate_coverage_neighbor_distance()`：覆盖率 + 邻域距离惩罚（当前主流程未默认调用）。

当前基础 DE 主流程采用的适应度近似为：

`fitness = coverage * cov_times / (average_move_distance + COV_DIST_CONSTANT)`

这意味着：覆盖率越高越好，平均移动距离越小越好。

### 3.4 差分演化主循环

每代主要步骤：

1. 选择变异策略（由 `operator_kind` 决定）；
2. 生成子代（变异 + 交叉）；
3. 越界修复（基本 DE 采用随机回置，JDE 采用边界截断）；
4. 父子个体分别评估；
5. 贪心选择（子代更优则替换父代）；
6. 更新历史最优解、记录收敛代与时间。

已实现策略包括（共 6 种，每种均有基本 DE 版本和 JDE 自适应版本）：

- `DE/Best/1`（operator_kind=0）
- `DE/Best/2`（operator_kind=1）
- `DE/Rand/1`（operator_kind=2）
- `DE/Rand/2`（operator_kind=3）
- `DE/Current-to-best/1`（operator_kind=4）
- `DE/Rand-to-best/1`（operator_kind=5）

JDE 版本的算子在每次变异前通过 tao1、tao2 参数自适应调整各个体的缩放因子 F 和交叉概率 CR。

### 3.5 后处理优化（非常关键）

DE 收敛后并不会直接结束，代码还有两步实用后优化：

1. **Discard（可不移动则不移动）**  
   采用两种方案并比较取优：方案一按位移距离从大到小排序逐一判断，方案二按节点编号顺序逐一判断。两种方案各自尝试将移动节点恢复到原位置，若覆盖率不降则取消其移动，最后选取覆盖率更优的方案结果，从而减少实际移动节点数。

2. **Exchange（交换匹配降距离）**  
   分两部分进行：第一部分在必须移动的节点间交换目标点分配，只要能降低总位移就交换；第二部分尝试将需移动节点的目标位置交给距离更近的不需移动节点（即需移动与不需移动节点之间的互换），在保持覆盖率的前提下进一步降低平均移动距离，最终得到更低平均移动距离的 `optimalmatch`。

这两步使结果更贴近工程应用，而不仅是"覆盖率最大化"。

## 4. 如何运行与演示给别人看

## 4.1 快速运行

- 启动 GUI：
  - `python main.py`
- 命令行验证（DE 与 JDE 各跑一次）：
  - `python script/validate.py`

> 建议在项目根目录执行，Python 3.10+ 更稳妥。

### 4.2 推荐演示流程（课堂/答辩可直接照做）

1. 打开 `main.py` 运行主界面；
2. 进入"参数设置"：
   - 网络来源选"从文件读出"并使用 `data/grid60.txt`；
   - 设定移动比例、种群规模、迭代次数；
3. 在"算子设置"中选一个变异算子（如 `DE/Best/1`）；
4. 点击"开始优化"，展示结果窗口：
   - 初始覆盖率 vs 优化后覆盖率；
   - 收敛代、收敛时间、总运行时间；
   - DE 平均移动距离、后处理优化距离、需移动节点数；
5. 打开 `result/output.txt`，展示关键结果可追溯；
6. 切换算法为 JDE 再跑一遍，对比 DE/JDE 指标差异。

### 4.3 演示时建议重点强调的指标

- `InitialCoverage` 与 `FinalCoverage`（效果核心）；
- `ConvergenceGeneration` 与 `RuntimeSec`（效率）；
- `NeedMovedNum` 与 `AverageMoveDistanceExchange`（工程成本）。

## 5. 给别人解答时可用的讲解框架（Q&A）

### Q1：这套算法优化的到底是什么？

答：在保证覆盖率尽量高的前提下，让移动节点的位移尽量小。主目标是覆盖，次目标是移动成本。

### Q2：为什么用 DE，而不是贪心或局部搜索？

答：节点位置优化是高维、非凸、离散+连续混合问题，DE 在这类全局搜索任务中鲁棒性较好，实现也相对简单；并且便于替换不同变异策略进行实验比较。

### Q3：代码里"网格覆盖"怎么判定？

答：以网格中心点为采样点，若中心点落在任一节点感知圆内即认为该网格被覆盖，最终按覆盖网格占比计算覆盖率。

### Q4：为什么还要做后处理（Discard + Exchange）？

答：DE 直接输出的目标点集合不一定对应最小位移分配。后处理能在不降低覆盖率的情况下减少移动节点数并降低平均移动距离，更符合工程实际。

### Q5：怎么证明结果可信？

答：同一输入下固定参数重复运行多次，统计平均覆盖率与方差；并对比不同算子、不同迭代次数、DE/JDE 两种算法，观察收敛质量与稳定性。

### Q6：JDE 和基本 DE 有什么区别？

答：JDE 是自适应差分演化，每个个体有独立的缩放因子 F 和交叉概率 CR，在迭代过程中通过 tao1、tao2 参数自动调整，无需人工调参；此外 JDE 使用纯覆盖率作为适应度函数，越界处理采用边界截断（而非基本 DE 的随机回置）。

## 6. 当前实现中的注意点

- `ui_main.py` 使用了 `os.startfile` 打开文件，这是 Windows 风格接口；在 Linux/macOS 上通常不会生效，建议改成平台兼容打开方式（不影响算法本身）。

## 7. 一句话总结

这个项目是一个"**可视化 + 可复现实验**"的混合 WSN 覆盖优化实现：用 DE/JDE 搜索移动节点布局，再通过后处理压缩移动成本，最后输出覆盖率与位移等可答辩指标。
